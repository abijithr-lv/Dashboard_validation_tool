### Why `ad_type` completeness is off

(Column name confirmed as `ad_type` via a real Databricks `UNRESOLVED_COLUMN`
error — not `media_type` as originally guessed; the dashboard display label
is "Media Type" but the underlying column is `ad_type`.)

The Glossary page documented four separate, still-recent source-system
changes all on this one dimension:
1. **Influencer (organic only)** has gone to **zero posts since week 2026-24** — leaving it in `expected_values` with completeness on would fail every week going forward.
2. **Influencer (organic + boosting)** moved from Golin's manual XLS to the **CreatorIQ API at week 2026-24** — only ~6 weeks of history on the new source as of the dashboard's current week (2026-30).
3. Golin → **Wavemakers** became the sole paid agency at week **2026-06**, after a gradual quarter-by-quarter handoff across all four Paid subtypes.
4. Organic moved from a manually maintained XLS to the **Sprinklr API starting 2026**, backfilled.

Rather than trust a single screenshot's snapshot against that much churn,
`completeness_check` is `false` for `ad_type` — re-enable it once these
migrations have fully settled and the value list is reconfirmed.

### Data quality notes carried into the YAML (not bugs — documented, not silently dropped)

- **`platform` has case-duplicate values** (`TikTok`/`TIKTOK`, `Instagram`/`INSTAGRAM`, `YouTube`/`YOUTUBE`, `LinkedIn`/`LINKEDIN`), almost certainly from two pipelines writing inconsistent casing. Both casings are kept in `expected_values` since both are real, current values — but this is worth normalizing upstream.
- **`gtm_segment_budget` intentionally mixes product segments, budget/program codes, and countries** in a single column (confirmed with the dashboard owner — "this is how it behaves," not a mislabeling).
- **`null`/`unknown` values are excluded from every dimension's `expected_values`** on purpose — they represent untagged rows, not real business categories. They won't be repeatedly flagged as "new" since they recur every week and fall inside the lookback window after the first run.

## Lookback window

`lookback_weeks: 6` — the trend check compares the current week against the
6-week average, and a dimension value has to be missing from all 6 of those
weeks before a reappearance counts as brand-new.

## Checks enabled for this dashboard

| Check | What it does here |
|---|---|
| freshness | Fails if the latest week in the dashboard isn't the run week |
| reconciliation | Dashboard totals must match source totals within tolerance, per metric |
| parts_sum | Per-`ad_type` subtotals must reconcile (confirmed: ad_type subtotals sum to the dashboard's grand total of 9,370,183,348 interactions) |
| trend_sanity | Whole-table change vs. the 6-week average must stay within ±50% |
| completeness | All expected values must appear for `platform`, `gtm_segment_budget`, `cloud_budget`, and `content_framework_pillar`; new ones get flagged. `ad_type` is currently exempt (see above) |
